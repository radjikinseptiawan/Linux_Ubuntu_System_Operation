# DNS Resolver
DNS Resolver adalah server atau sistem perantara (biasanya disediakan ISP atau pihak ketiga) yang berfungsi 
menerjemahkan nama domain yang manusia baca menjadi alamat IP mesin untuk mengakses situs web

Secara Sederhana DNS resolver bisa dianggap perantara antara pengguna internet dan server DNS, proses ini 
berlangsung sangat cepat, meskipun melibatkan berbagai server seperti TLD, root, hingga authoritative name server. 
Umumnya seluruh rangkaian tersebut bisa selesai hanya dalam beberapa milidetik

# Cara Kerja DNS Resolver
Proses ini sebenarnya terdiri dari beberapa tahapan berurutan yang tampak rumit, namun kenyataan nya berlangsung sangat
cepat dan hampir tidak terasa. Berikut penjelasan nya 
1. Pertama, pengguna internet melalui browser mengirim permintaan DNS ke resolver terlebih dahulu
2. Setelah itu, resolver memeriksa cache lokal. jika domain tersebut sudah pernah diakss dan datanya masih berlaku, maka 
   resolver langsung memberikan jawaban tanpa harus meminta ke server lain
3. Jika informasi tidak ada di cache maka resolver akan menghubungin root name server. root tidak memberikan IP address secara langsung
   tetapi menunjuk resolver ke TLD name server yang sesuai
4. Resolver kemudian meminta data ke TLD name server, yang akan mengarahkan ke authoritative name server dari domain terkait
5. Dari authoritative name server inilah resolver memperoleh alamat IP  domain yang dimaksud setelah itu, IP terkirim kembali ke browser
   dan juga tersimpan di cache resolver agar akses berikutnya lebih cepat
# Jenis Query DNS
Selain memahami cara kerja anda perlu juga mengetahui jenis query yang resolver gunakan. jenis query inilah yang memengaruhi bagaimana 
resolver mencari data serta seberapa cepat jawaban bisa diberikan. di antaranya sebagai berikut
- Recursive Query = resolver akan menuntaskan seluruh permintaan klien hingga berhasil mendapatkan jawaban lengkap. 
- Iterative Query = resolver hanya memberi jawaban sementara atau referensi, proses tersebut berlanjut ke server sampai menemukan jawaban
- Non-Revursive Query = Resolver bisa langsung menjawab dari cache tanpa harus bertanya ke server lain
